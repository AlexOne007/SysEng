# 220. Архитектура системы

## 1. Общие сведения
Система предназначена для автоматизации предварительного заказа еды в фастфуде.

## 2. Компоненты
| Компонент | Технология         | Назначение                     |
|-----------|--------------------|--------------------------------|
| Клиент    | Браузер            | Отображение интерфейса         |
| Web UI    | HTML/CSS/JS        | Взаимодействие с пользователем |
| Backend   | Java + Spring Boot | Бизнес-логика, REST API        |
| Database  | Oracle 21c         | Хранение данных                |

# 230. Описание сервисов (API)

## 1. Сервис товаров: `/api/products`
| Метод | Описание         | Параметры | Ответ                                                 |
|-------|------------------|-----------|-------------------------------------------------------|
| GET   | Получить каталог | —         | `[{"productId":1,"name":"Чизбургер","price":149.90}]` |

## 2. Сервис заказов: `/api/orders`
| Метод  | Описание       | Параметры                   | Ответ                                      |
|--------|----------------|-----------------------------|--------------------------------------------|
| POST   | Создать заказ  | `{user, items}`             | `{"orderId":123,"status":"NEW"}`           |
| GET    | Получить заказ | `/{id}`                     | `{"orderId":123,"status":"PREPARING",...}` |
| PUT    | Сменить статус | `/{id}/status?status=READY` | `{"orderId":123,"status":"READY"}`         |
| DELETE | Отменить заказ | `/{id}`                     | `204 No Content`                           |

## 3. Формат данных
### Запрос на создание заказа:
```json
{
  "user": 1,
  "items": [
    {"productId": 1, "quantity": 2, "price": 149.90}
  ]
}
```

### Ответ
```json
{
  "orderId": 2, 
  "status": "NEW", 
  "createdAt": "2026-03-08T14:30:00"
}
```

# 240. Интерфейс пользователя
### 1 Каталог товаров
- Отображение карточек товаров (название, цена)
- Кнопка «В корзину» для каждого товара

### 2 Корзина
- Список выбранных товаров с возможностью изменить количество
- Итоговая сумма
- Кнопки «Оформить» / «Очистить»

### 3 Оформление заказа
- Поля: Имя*, Телефон*,
- Валидация обязательных полей
- Кнопка «Подтвердить»

### 4 Статус заказа
- Номер заказа и текущий статус
- Кнопка «Отменить» (активна только для статуса «Новый»)

### 5 Админ-панель
- Таблица активных заказов
- Управление статусами заказов
- Управление меню (добавление/удаление товаров)
